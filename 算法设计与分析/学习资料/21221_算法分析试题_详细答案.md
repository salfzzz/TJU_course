# 21221_算法分析试题详细答案

本文对应 `21221_算法分析试题.pdf`，按题目顺序给出答案和步骤详解。

## 一、简答

### 1. 渐近记号的含义

渐近记号用于描述当问题规模 `n` 趋于无穷大时，算法复杂度函数的增长趋势。它忽略常数因子和低阶项，只关注主要增长速度。

常用记号如下：

1. `O(g(n))`：渐近上界。

   若存在正常数 `c` 和 `n0`，使得当 `n >= n0` 时，有

   ```text
   0 <= f(n) <= c g(n)
   ```

   则称 `f(n)=O(g(n))`。

2. `Omega(g(n))`：渐近下界。

   若存在正常数 `c` 和 `n0`，使得当 `n >= n0` 时，有

   ```text
   0 <= c g(n) <= f(n)
   ```

   则称 `f(n)=Omega(g(n))`。

3. `Theta(g(n))`：渐近紧确界。

   若同时有

   ```text
   f(n)=O(g(n))
   f(n)=Omega(g(n))
   ```

   则称 `f(n)=Theta(g(n))`。

一句话：`O` 表示不超过某个增长级别，`Omega` 表示不低于某个增长级别，`Theta` 表示增长级别正好相同。

### 2. 用 Master 方法解递归方程

题目：

```text
T(n) = 5T(n/5) + n lg n
```

与 Master 定理标准形式比较：

```text
T(n) = aT(n/b) + f(n)
```

可得：

```text
a = 5
b = 5
f(n) = n lg n
```

计算：

```text
n^(log_b a) = n^(log_5 5) = n
```

而：

```text
f(n) = n lg n = Theta(n log^1 n)
```

属于 Master 定理第二种情况：

```text
f(n) = Theta(n^(log_b a) log^k n)
```

其中 `k=1`。

所以：

```text
T(n) = Theta(n log^(k+1) n)
     = Theta(n log^2 n)
```

答案：

```text
T(n) = Theta(n lg^2 n)
```

### 3. P、NP、NP 难、NP 完全问题

1. 类 `P`

   能够在多项式时间内求解的问题集合。

   例如排序、最短路、最小生成树等。

2. 类 `NP`

   给定一个候选解后，能够在多项式时间内验证该解是否正确的问题集合。

   注意：`NP` 不是“非多项式”的意思，而是“可在多项式时间内验证”。

3. `NP` 难问题

   如果所有 `NP` 问题都可以在多项式时间内归约到问题 `Q`，则称 `Q` 是 `NP` 难问题。

   `NP` 难问题不一定属于 `NP`。

4. `NP` 完全问题

   如果问题 `Q` 同时满足：

   ```text
   Q 属于 NP
   Q 是 NP 难问题
   ```

   则称 `Q` 是 `NP` 完全问题。

关系图可以这样表示：

```text
                 NP-hard
        ┌─────────────────────┐
        │                     │
        │        NP           │
        │   ┌─────────────┐   │
        │   │   ┌─────┐   │   │
        │   │   │  P  │   │   │
        │   │   └─────┘   │   │
        │   │             │   │
        │   │   NPC       │   │
        │   └─────────────┘   │
        │                     │
        └─────────────────────┘
```

其中：

```text
P ⊆ NP
NPC = NP ∩ NP-hard
NP-hard 可以包含 NP 之外的问题
```

目前尚未证明 `P=NP` 或 `P!=NP`。

## 二、算法分析

### 1. 插入操作的最好、最坏、平均复杂度

题目代码：

```cpp
for (int i = n - 1; i >= 0 && x < a[i]; i--)
    a[i + 1] = a[i];

a[i + 1] = x;
```

这段代码是在一个已有序数组中插入元素 `x`。

循环从右往左扫描，只要 `x < a[i]`，就把 `a[i]` 后移一位。

#### 最好情况

如果：

```text
x >= a[n-1]
```

说明 `x` 比数组中最大元素还大，直接插到最后即可。

循环条件第一次判断就失败，不发生元素移动。

所以：

```text
T_best(n) = Theta(1)
```

#### 最坏情况

如果：

```text
x < a[0]
```

说明 `x` 比所有元素都小，需要把全部 `n` 个元素都后移一位。

移动次数为 `n`。

所以：

```text
T_worst(n) = Theta(n)
```

#### 平均情况

假设 `x` 插入到每个位置的概率相同。

可能移动：

```text
0, 1, 2, ..., n
```

平均移动次数为：

```text
(0 + 1 + 2 + ... + n) / (n + 1)
= n / 2
```

所以：

```text
T_avg(n) = Theta(n)
```

答案：

```text
最好情况：Theta(1)
最坏情况：Theta(n)
平均情况：Theta(n)
```

### 2. Hanoi 问题复杂度

题目代码：

```cpp
void Hanoi(A, C, n) {
    if (n == 1)
        move(A, C);
    else {
        Hanoi(A, B, n - 1);
        move(A, C);
        Hanoi(B, C, n - 1);
    }
}
```

设移动 `n` 个盘子的操作次数为 `T(n)`。

当 `n=1` 时：

```text
T(1) = 1
```

当 `n>1` 时：

先把 `n-1` 个盘子从 A 移到 B，需要 `T(n-1)` 次。

再把最大盘子从 A 移到 C，需要 `1` 次。

最后把 `n-1` 个盘子从 B 移到 C，需要 `T(n-1)` 次。

所以：

```text
T(n) = 2T(n-1) + 1
```

展开：

```text
T(n) = 2T(n-1) + 1
     = 2[2T(n-2)+1] + 1
     = 4T(n-2) + 2 + 1
     = 8T(n-3) + 4 + 2 + 1
     = ...
     = 2^(n-1)T(1) + 2^(n-2) + ... + 2 + 1
     = 2^(n-1) + 2^(n-1) - 1
     = 2^n - 1
```

因此：

```text
T(n) = Theta(2^n)
```

## 三、贪心法

题意：有 `n` 个任务，任务 `i` 需要 `t_i` 个时间单位。要求安排任务顺序，使平均完成时间 ACT 最小。

贪心策略：每一步选择剩余任务中处理时间最短的任务。

这就是“短作业优先”策略。

### 1. 伪代码和复杂度

可以直接排序：

```text
Greedy-ACT(t[1..n])
    按照处理时间 t_i 从小到大排序所有任务
    current = 0
    sum = 0
    for i = 1 to n
        current = current + t[i]
        c[i] = current
        sum = sum + c[i]
    return sum / n
```

如果用排序实现：

```text
排序复杂度：O(n log n)
计算完成时间：O(n)
总复杂度：O(n log n)
```

如果严格按照“每一步从剩余任务中选最短任务”的方式，不预排序，而是每轮扫描剩余任务，则复杂度为：

```text
O(n^2)
```

考试中两种写法都可以，但推荐写排序版本，复杂度更好。

### 2. 最优性证明

使用交换论证。

设某个最优任务序列中，存在两个相邻任务 `i` 和 `j`，并且：

```text
t_i > t_j
```

也就是说，较长的任务 `i` 排在较短的任务 `j` 前面。

设在执行任务 `i` 和 `j` 之前，已经花费时间为 `S`。

原顺序为：

```text
i, j
```

这两个任务对完成时间总和的贡献是：

```text
(S + t_i) + (S + t_i + t_j)
= 2S + 2t_i + t_j
```

交换后顺序为：

```text
j, i
```

贡献变成：

```text
(S + t_j) + (S + t_j + t_i)
= 2S + 2t_j + t_i
```

比较两者：

```text
原贡献 - 新贡献
= (2S + 2t_i + t_j) - (2S + 2t_j + t_i)
= t_i - t_j
```

因为 `t_i > t_j`，所以：

```text
t_i - t_j > 0
```

说明交换后完成时间总和变小，ACT 也变小。

因此，在最优序列中不可能存在“长任务排在短任务前面”的相邻逆序。

所以最优序列必须按处理时间非递减排序。

因此，贪心算法最优。

## 四、分治法

题意：对数组 `A[1:n]` 排序，使用快速排序，每次选择当前待排序子数组的第 3 个元素作为支点。

若当前子数组长度不足 3，可以选择第一个元素作为支点，或者直接用普通方式处理。这里给出一种完整伪代码。

### 伪代码

```text
QuickSort(A, p, r)
    if p >= r
        return

    if r - p + 1 >= 3
        pivotIndex = p + 2
    else
        pivotIndex = p

    swap A[pivotIndex] and A[r]
    q = Partition(A, p, r)
    QuickSort(A, p, q - 1)
    QuickSort(A, q + 1, r)
```

其中 `Partition` 为：

```text
Partition(A, p, r)
    x = A[r]
    i = p - 1
    for j = p to r - 1
        if A[j] <= x
            i = i + 1
            swap A[i] and A[j]
    swap A[i + 1] and A[r]
    return i + 1
```

### 复杂度分析

每次划分需要线性时间：

```text
Theta(n)
```

#### 最好情况

如果每次支点都能把数组大致分成相等的两部分：

```text
T(n) = 2T(n/2) + Theta(n)
```

由 Master 定理：

```text
T(n) = Theta(n log n)
```

#### 最坏情况

如果每次选择的第 3 个元素恰好是当前子数组的最大值或最小值，则每次只能划分出一个规模为 `0` 的子数组和一个规模为 `n-1` 的子数组。

此时：

```text
T(n) = T(n - 1) + Theta(n)
```

展开：

```text
T(n) = n + (n-1) + (n-2) + ... + 1
     = Theta(n^2)
```

答案：

```text
最好情况：Theta(n log n)
最坏情况：Theta(n^2)
```

## 五、动态规划法

题设：`g(i,x)` 表示剩余物品为 `1,2,...,i`，剩余背包容量 `c=x` 的 0/1 背包问题最大效益值。

### 1. 递归关系

边界条件：

```text
g(0,x) = 0
```

当没有物品可选时，最大效益为 0。

对于第 `i` 个物品，只有两种选择：

1. 不选第 `i` 个物品，效益为：

   ```text
   g(i-1,x)
   ```

2. 选第 `i` 个物品，前提是 `x >= w_i`，效益为：

   ```text
   g(i-1, x-w_i) + p_i
   ```

所以递归关系为：

```text
若 x < w_i：
g(i,x) = g(i-1,x)

若 x >= w_i：
g(i,x) = max{ g(i-1,x), g(i-1,x-w_i) + p_i }
```

### 2. 元组法计算 `g(4,20)`

题目：

```text
n = 4
c = 20
p = (2,5,8,1)
w = (10,15,6,9)
```

物品列表：

```text
物品1：w1=10, p1=2
物品2：w2=15, p2=5
物品3：w3=6,  p3=8
物品4：w4=9,  p4=1
```

元组 `(weight, profit)` 表示某种选择方案的总重量和总价值。

只保留非支配元组。若存在两个元组：

```text
(w1, p1), (w2, p2)
```

并且：

```text
w1 <= w2 且 p1 >= p2
```

则 `(w2,p2)` 被支配，可以删除。

#### 处理物品 1

不选物品 1：

```text
(0,0)
```

选物品 1：

```text
(10,2)
```

所以：

```text
P1 = {(0,0), (10,2)}
```

#### 处理物品 2

由 `P1` 生成选物品 2 的元组：

```text
(0+15, 0+5) = (15,5)
(10+15, 2+5) = (25,7)  超过容量20，删除
```

合并后：

```text
P2 = {(0,0), (10,2), (15,5)}
```

#### 处理物品 3

由 `P2` 生成选物品 3 的元组：

```text
(0+6, 0+8) = (6,8)
(10+6, 2+8) = (16,10)
(15+6, 5+8) = (21,13) 超过容量20，删除
```

合并：

```text
(0,0), (10,2), (15,5), (6,8), (16,10)
```

删除被支配元组：

```text
(10,2) 被 (6,8) 支配
(15,5) 被 (6,8) 支配
```

所以：

```text
P3 = {(0,0), (6,8), (16,10)}
```

#### 处理物品 4

由 `P3` 生成选物品 4 的元组：

```text
(0+9, 0+1) = (9,1)
(6+9, 8+1) = (15,9)
(16+9, 10+1) = (25,11) 超过容量20，删除
```

合并：

```text
(0,0), (6,8), (16,10), (9,1), (15,9)
```

删除被支配元组：

```text
(9,1) 被 (6,8) 支配
```

因此：

```text
P4 = {(0,0), (6,8), (15,9), (16,10)}
```

容量不超过 20 时，最大价值是：

```text
g(4,20) = 10
```

#### 回溯得到最优解

最终最优元组是：

```text
(16,10)
```

它在 `P4` 中直接来自 `P3`，不是加上物品 4 得到的，所以：

```text
x4 = 0
```

在 `P3` 中，`(16,10)` 是由：

```text
(10,2) + 物品3(6,8)
```

得到的，所以：

```text
x3 = 1
```

回到 `P2` 中的 `(10,2)`，它不是加上物品 2 得到的，所以：

```text
x2 = 0
```

`(10,2)` 是由物品 1 得到的，所以：

```text
x1 = 1
```

最优解：

```text
x = (1,0,1,0)
```

即选择物品 1 和物品 3。

总重量：

```text
10 + 6 = 16 <= 20
```

总价值：

```text
2 + 8 = 10
```

所以：

```text
最优值 = 10
最优解 = (1,0,1,0)
```

## 六、回溯-分支限界法

城市编号：

```text
1 = 北京
2 = 上海
3 = 广州
4 = 南京
```

原始距离矩阵为：

```text
A =
∞    500   600   100
100  ∞     800   500
1000 200   ∞     2000
400  400   100   ∞
```

其中对角线的 `0` 改为 `∞`，表示不能从城市自己到自己。

### 1. 回溯法

#### 限界条件

显式约束：

```text
每个城市恰好访问一次，最后回到起点城市北京。
```

隐式约束：

```text
当前路径长度 currentCost < bestc
```

如果当前路径长度已经不小于当前最优值 `bestc`，则该分支不可能得到更优解，可以剪枝。

#### 按城市编号顺序展开状态空间树

初始：

```text
bestc = ∞
```

从北京 `1` 出发。

状态空间树如下，`S` 表示当前已经走过的路径长度。

```text
1 北京
├─ 2 上海, S=500
│  ├─ 3 广州, S=1300
│  │  └─ 4 南京, S=3300, 回到1加400，总长3700
│  │      bestc = 3700
│  └─ 4 南京, S=1000
│     └─ 3 广州, S=1100, 回到1加1000，总长2100
│         bestc = 2100
├─ 3 广州, S=600
│  ├─ 2 上海, S=800
│  │  └─ 4 南京, S=1300, 回到1加400，总长1700
│  │      bestc = 1700
│  └─ 4 南京, S=2600
│      因为 2600 >= bestc=1700，剪枝
└─ 4 南京, S=100
   ├─ 2 上海, S=500
   │  └─ 3 广州, S=1300, 回到1加1000，总长2300
   │      bestc 仍为 1700
   └─ 3 广州, S=200
      └─ 2 上海, S=400, 回到1加100，总长500
          bestc = 500
```

最终得到最优回路：

```text
北京 -> 南京 -> 广州 -> 上海 -> 北京
```

即：

```text
1 -> 4 -> 3 -> 2 -> 1
```

总长度：

```text
北京 -> 南京 = 100
南京 -> 广州 = 100
广州 -> 上海 = 200
上海 -> 北京 = 100
```

所以：

```text
最优值 = 100 + 100 + 200 + 100 = 500
```

### 2. 分支限界法：规约矩阵法

#### 第一步：根结点规约

原始矩阵：

```text
A =
∞    500   600   100
100  ∞     800   500
1000 200   ∞     2000
400  400   100   ∞
```

每一行减去本行最小值。

第 1 行最小值为 `100`。

第 2 行最小值为 `100`。

第 3 行最小值为 `200`。

第 4 行最小值为 `100`。

行规约总和：

```text
h = 100 + 100 + 200 + 100 = 500
```

规约后：

```text
A' =
∞    400   500   0
0    ∞     700   400
800  0     ∞     1800
300  300   0     ∞
```

每一列已经都有 0，所以列规约值为 0。

根结点下界：

```text
δ(1) = 500
```

#### 第二步：从北京 1 出发扩展

公式：

```text
δ(子结点) = δ(父结点) + 规约矩阵中的边代价 + 新矩阵规约值
```

##### 选择 1 -> 2

在 `A'` 中：

```text
a'12 = 400
```

选边 `1 -> 2` 后，删除第 1 行、第 2 列，并禁止 `2 -> 1`。继续规约，新规约值：

```text
h' = 1200
```

所以：

```text
δ(2) = 500 + 400 + 1200 = 2100
```

##### 选择 1 -> 3

在 `A'` 中：

```text
a'13 = 500
```

继续规约得到：

```text
h' = 700
```

所以：

```text
δ(3) = 500 + 500 + 700 = 1700
```

##### 选择 1 -> 4

在 `A'` 中：

```text
a'14 = 0
```

继续规约得到：

```text
h' = 0
```

所以：

```text
δ(4) = 500 + 0 + 0 = 500
```

第一层状态空间树：

```text
             1 北京
             δ=500
       /        |        \
  2 上海     3 广州     4 南京
  δ=2100    δ=1700     δ=500
```

下界最小的是 `1 -> 4`，所以展开结点 4。

#### 第三步：展开 1 -> 4

选了 `1 -> 4` 后，剩余规约矩阵可写为：

```text
B =
0    ∞    700
800  0    ∞
∞    300  0
```

当前在城市 4，可以去城市 2 或城市 3。

##### 选择 4 -> 2

规约矩阵中对应值为：

```text
300
```

继续规约得到：

```text
h' = 1500
```

所以：

```text
δ(1 -> 4 -> 2) = 500 + 300 + 1500 = 2300
```

##### 选择 4 -> 3

规约矩阵中对应值为：

```text
0
```

继续规约得到：

```text
h' = 0
```

所以：

```text
δ(1 -> 4 -> 3) = 500 + 0 + 0 = 500
```

状态空间树变为：

```text
             1 北京
             δ=500
       /        |        \
  2 上海     3 广州     4 南京
  δ=2100    δ=1700     δ=500
                         /      \
                    2 上海     3 广州
                    δ=2300     δ=500
```

继续展开下界最小的 `1 -> 4 -> 3`。

#### 第四步：展开 1 -> 4 -> 3

当前路径：

```text
1 -> 4 -> 3
```

剩余未访问城市只有上海 `2`，所以只能走：

```text
3 -> 2
```

得到：

```text
δ(1 -> 4 -> 3 -> 2) = 500
```

最后从上海回到北京：

```text
2 -> 1
```
