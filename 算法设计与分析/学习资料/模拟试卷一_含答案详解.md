## 答案与详解

### 一、算法分析与渐进复杂度

1. 算法复杂性分析主要研究算法运行时需要多少时间、占用多少空间。常用方法有解析法和实验测量法；常用指标是时间复杂度和空间复杂度。用途是比较不同算法的效率，在问题规模变大时判断哪个算法更适合。

2. 递归式：

   1. `T(n)=8T(n/2)+n^2`。这里 `a=8,b=2`，`n^(log_b a)=n^3`，而 `f(n)=n^2` 比 `n^3` 小一个多项式量级，所以 `T(n)=Theta(n^3)`。
   2. `T(n)=4T(n/2)+n^2 log n`。这里 `a=4,b=2`，`n^(log_b a)=n^2`，`f(n)=n^2 log n`，属于 Master 定理第二类的扩展形式，`k=1`，所以 `T(n)=Theta(n^2 log^2 n)`。
   3. `T(n)=T(n-1)+3n`。展开得 `T(n)=T(1)+3(2+3+...+n)`，因此 `T(n)=Theta(n^2)`。

3. 对固定的 `j`，`while` 中 `k` 依次为 `1,2,4,...`，执行次数为 `floor(log2 j)+1`，也就是 `Theta(log j)`。

   因此总次数为：

   ```text
   sum_{i=1}^n sum_{j=1}^i Theta(log j)
   ```

   对固定的 `i`，内部求和为 `Theta(i log i)`；再对 `i=1...n` 求和，得到 `Theta(n^2 log n)`。

### 二、分治法

1. 分治思想：把数组分成左右两半，分别计算左半部分逆序数、右半部分逆序数，再在归并两个有序数组时计算跨越左右两半的逆序数。

```text
CountInv(A, l, r):
    if l >= r: return 0
    mid = (l+r)/2
    left = CountInv(A, l, mid)
    right = CountInv(A, mid+1, r)
    cross = MergeAndCount(A, l, mid, r)
    return left + right + cross
```

归并时，如果右边当前元素小于左边当前元素，说明左边从当前元素到末尾都比它大，这些都构成逆序。

2. 对 `A=[4,1,3,2,5]`：

   左半 `[4,1,3]` 中有 `(4,1),(4,3)`，共 2 个。

   右半 `[2,5]` 中没有逆序。

   合并有序的 `[1,3,4]` 和 `[2,5]` 时：

   - `1<2`，不产生逆序。
   - `2<3`，右边的 `2` 比左边剩余的 `3,4` 都小，产生 `(3,2),(4,2)`，共 2 个。

   总逆序数为 `2+0+2=4`。

3. 递归式为 `T(n)=2T(n/2)+Theta(n)`，由 Master 定理得 `T(n)=Theta(n log n)`。

### 三、贪心法

1. 贪心策略：按活动开始时间从小到大排序。每来一个活动，把它放入结束时间最早且不冲突的教室；若所有教室都冲突，则新开一个教室。

2. 按开始时间排序后依次处理：

| 活动 | 安排 |
|---|---|
| `[1,4)` | 教室 1 |
| `[2,5)` | 教室 2 |
| `[4,7)` | 教室 1，因为教室 1 在 4 时刻空出 |
| `[5,9)` | 教室 2，因为教室 2 在 5 时刻空出 |
| `[6,8)` | 教室 3，因为 6 时刻教室 1、2 都未空出 |

最少需要 3 个教室。

3. 证明：任一时刻同时进行的活动数，是任何安排都无法低于的教室数下界。该算法每次只有在所有已有教室都与当前活动冲突时才新开教室，这说明当前活动开始时已有教室中的活动都还没结束，因此确实存在这么多个活动同时进行。新开教室的次数不会超过这个下界，所以算法最优。

### 四、动态规划

1. 设 `m[i][j]` 表示计算 `Ai...Aj` 所需的最少数乘次数。则：

```text
m[i][i] = 0
m[i][j] = min_{i <= k < j} { m[i][k] + m[k+1][j] + r[i-1] * r[k] * r[j] }
```

其中 `k` 表示最后一次乘法把矩阵链分成 `Ai...Ak` 和 `A(k+1)...Aj` 两部分。

2. 计算过程：

长度为 2：

```text
m[1,2]=5*10*3=150
m[2,3]=10*3*12=360
m[3,4]=3*12*5=180
m[4,5]=12*5*50=3000
```

长度为 3：

```text
m[1,3]=min(0+360+5*10*12, 150+0+5*3*12)=330
m[2,4]=min(0+180+10*3*5, 360+0+10*12*5)=330
m[3,5]=min(0+3000+3*12*50, 180+0+3*5*50)=930
```

长度为 4：

```text
m[1,4]=min(0+330+5*10*5, 150+180+5*3*5, 330+0+5*12*5)=405
m[2,5]=min(0+930+10*3*50, 360+3000+10*12*50, 330+0+10*5*50)=2430
```

长度为 5：

```text
m[1,5]=min(
  0+2430+5*10*50,
  150+930+5*3*50,
  330+3000+5*12*50,
  405+0+5*5*50
)=1655
```

最少数乘次数为 `1655`。最优加括号方式为：

```text
(((A1A2)(A3A4))A5)
```

### 五、回溯法与分支限界法

1. 先按单位价值从大到小排序，便于计算上界：

| 物品 | 重量 | 价值 | 单位价值 |
|---|---:|---:|---:|
| 4 | 5 | 8 | 1.60 |
| 1 | 2 | 3 | 1.50 |
| 2 | 3 | 4 | 1.33 |
| 3 | 4 | 5 | 1.25 |

回溯法的基本限界条件：

```text
当前重量 cw > c 时剪枝
当前价值 cp + 剩余物品可达到的上界 <= bestp 时剪枝
```

搜索主要过程：

```text
选物品4: cw=5, cp=8
    选物品1: cw=7, cp=11 -> 当前最优 bestp=11
    不选物品1: 上界不超过 11，可剪枝
不选物品4:
    即使装入物品1、2，价值为 7；再部分估计物品3，上界也不超过 11，可剪枝
```

所以最优方案是选择物品 1 和物品 4。按原顺序写为：

```text
x = (1,0,0,1)
```

最优价值为 `3+8=11`，总重量为 `2+5=7`。

2. LC 分支限界法使用最大上界优先扩展。上界函数可用连续背包估计：

```text
bound(node):
    从当前层后面的物品开始，按单位价值从高到低尽量装入
    能整件装就整件装，最后一个若装不下则装入一部分
```

主要结点：

| 结点 | 决策前缀（排序后） | cw | cp | bound |
|---|---|---:|---:|---:|
| 根 | `()` | 0 | 0 | 11 |
| A | `(1)` 选物品4 | 5 | 8 | 11 |
| B | `(0)` 不选物品4 | 0 | 0 | 9.50 |
| C | `(1,1)` 选物品4、1 | 7 | 11 | 11 |

结点 C 已达到容量，得到可行解 `bestp=11`。其他结点的上界不超过 11 或继续扩展无法超过 11，因此剪枝。

最终最优解仍为：

```text
x = (1,0,0,1), bestp = 11
```
