# AGENTS.md

## 本目录任务背景
- 这是《算法设计与分析》的课件与往年题目录。
- 每个课件文件名前都有两位数字编号，例如 `02.第二节-算法分析.pdf`。用户说“02”时，通常指文件名前缀为 `02` 的课件。
- 用户会逐页学习课件，常要求“下一页”“讲第几页”“下三页一起讲”。必须严格按 PDF 实际页码处理。
- 每个往年题文件的文件名，命名方式都为日期+学期的数字编号，例如：`17182_算法分析.pdf`。
- 用户询问往年题的题目时，会先指明对应的那套往年题中的第几题，如：`17182的第3题`。


## 讲解方式
- 用户让讲多少页，就只讲多少页；不要提前讲下一页内容。
- 每次回答前先确认正在讲的 PDF 实际页码和页面标题。
- 对课件上稍微难理解的句子，要逐句拆开讲，不要只概括。
- 整体使用通俗语言讲解，目标是让七十岁老人也能听懂。
- 遇到考试可能会考的重点，先用通俗语言讲明白，再额外给一版较专业、适合简答题的标准表述。
- 不要假设用户有算法基础。遇到术语要立刻用白话解释。
- 可以适当总结，但不能用总结代替细讲。
- 对于课件里涉及的具体算法实现步骤，按每一步实现步骤，结合图例讲解

## 算法讲解
- 对于某一算法的具体实例，每一步都要拆解，并分析清楚。
- 讲解完一个实例以后，给出关于该实例的变种题目，并按上述同样的方法详细讲解。
- 在用户询问具体算法的步骤时，严格参照课件上的过程来讲解。
- 如果用户询问的是课件中未具体说明的步骤，则详细补充该步骤的实现过程，并配套`讲解方式`中的方法讲解

## PDF 阅读要求
- 不能只依赖 `pdftotext` 或文字层，因为 PDF 的图、箭头、表格、右侧结构图、英文标注可能提取不出来。
- 对包含图示、流程、箭头、表格、结构图的页面，应渲染截图并根据截图补充讲解。
- 如果用户指出漏讲图，要承认并补讲，不要辩解。
- 中间产物遵守全局 `AGENTS.md` 规则，放在 `codex_proc/` 下。

## 推荐工作流程
1. 根据用户指定的课件编号或文件名定位 PDF。
2. 根据用户指定页码抽取对应页文字。
3. 同时或必要时渲染页面截图，检查是否有图、箭头、表格、英文标注。
4. 严格只讲用户要求的页数。
5. 逐句讲解难点，英文缩写每次重新展开。
6. 对考试重点额外给出专业表述。
7. 用通俗比喻收尾，总结本页最重要的一句话。

## 讲解往年题步骤
1. 当用户提到往年题时，要阅读往年题里的题目，结合之前给用户讲解的知识点解答用户疑问
2. 解答之后还要举出几道相似例题，供用户练习使用。
3. 用户对题目提出的任何反问，都先按照课件上的步骤解答，再用最浅显，好理解的语言讲解。
